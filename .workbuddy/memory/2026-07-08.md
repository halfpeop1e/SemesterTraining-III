## 仿真跨页面中断 bug 修复（21:39~22:10）

### 用户反馈
- "在调度控制点启动 → 切到线路信号，T1没动 → 切回调度，启动被中断了"

### 根因诊断
- 调度页用自身组件内 `setInterval`(500ms) 调 `stepSimulation()` 推仿真；interval + `isRunning` 绑在 Dispatch 生命周期上
- App 用 `module` 状态硬切换页面(App.tsx)，切走即卸载 Dispatch → interval 清掉 → 后端冻结
- 后端**无**任何调度器(ScheduledExecutor/@Scheduled/Timer grep 全空)：只在被 stepSimulation 调用时才前进，不自跑
- LineSignal 被动 getSnapshot，后端冻结则 T1 不动；切回 Dispatch 重新挂载 → isRunning=false → 显示「启动」= "被中断"

### 修复方案（用户选：共享 Provider）
1. 新增 `src/context/SimulationContext.tsx`：SimulationProvider 持有 snapshot/isRunning/loading/error/speedIndex + start/stop/step/setSpeed/setError，interval 推进循环在 Provider 内
2. `App.tsx`：用 `<SimulationProvider>` 包裹 `<PageComponent/>`（位于页面切换层之上，切 module 不卸载）
3. `Dispatch/index.tsx`：移除本地 autoRef/refresh/start/stop/step + 三个相关 useEffect/状态，改为 `useSimulation()` 消费；速度按钮 setSpeedIndex→setSpeed
4. `LineSignal/index.tsx`：移除 getSnapshot 导入 + tickSim + 800ms 轮询 + simRef；改为 `useSimulation()` 读 snapshot，`contextTrains` useMemo 映射 + `recomputeMa` 随快照重算 MA；降级/刷新MA 按钮改调 setShowDegraded/recomputeMa；删 showDegradedRef/useRef

### 模块边界（仍遵守）
- 启动/停止/推进按钮只存在于调度页；LineSignal 仍只读快照，不 start/step
- 纯前端改动，不动后端、不动 compute()

### 验证（Playwright，dev server :5173 代理→:8080）
