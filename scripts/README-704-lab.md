# 老师台架 704 / 司机台联调速查

详细对照见：[`docs/protocol/老师台架接口映射.md`](../docs/protocol/老师台架接口映射.md)

## 设备角色（协议文档）

| IP | 角色 | 端口 | 我们做什么 |
|----|------|------|------------|
| `192.168.100.123` | **PLC 输入/回写** | TCP `8001/8002/8003` | ✅ 读 46B 手柄帧；⚠️ 26B/28B 回写待在实体台架验收 |
| `192.168.100.121` | **网络屏** | TCP **`8888`** | ✅ 写 572B；✅ 收到 572B 显示回显；❌ 26B 牵引切除命令未验收 |
| `192.168.100.122` | **信号屏** | TCP **`9999`** | ✅ 写 66B |
| `192.168.100.124` | **视景**（口头） | UDP（端口待确认） | ❌ 未做 |

> 2026-07-12 现场采集已确认：`.121:8888` 为网络屏，`.122:9999` 为信号屏。

## 今天就能用：只读 PLC 控仿真

```powershell
ping 192.168.100.123
python scripts/tcp704_live_demo.py --reconnect
```

1. 后端 `mvn spring-boot:run`。默认所有台架输出均关闭。
2. 在车载前端对目标车点击“上线并等待发车”。这会为当前 `trainId` 建立后端可控上下文，不需要另建一套列车。
3. Protocol704 面板 Connect（同一 `trainId`）
4. 推手柄 → 看仿真是否动

## 2026-07-12 现场样本结论

- PLC 输入：`46B`，约 `100ms` 一帧。本次样本实际覆盖前/零方向、牵引 0–100%、钥匙和 ATO 启动；门、制动、EB 等尚未操作验证。
- PLC 输出：朋友系统曾写入 `28B`，`totalLen=28`、`dataLen=4`；这是其真实台架操作记录，但 PLC 的后续 46B 帧没有回显写入速度，故不能视为已验收的老师接口。后端默认按老师协议总表发送 26B，28B 仅可显式开启为对照变体。
- 网络屏：`572B` 上行后逐帧收到同长度回显，说明 TCP 连接和帧长已在实物环境中跑通；字段显示效果仍须到现场逐项核验。
- 信号屏：`66B` 稳定发送；此次记录没有回包，不能凭连接成功推断每个显示字段都已正确。
- 视景：采集到 `1556B` UDP 帧，但目标是 `host.docker.internal:8303`，不是老师 `.124` 的已确认地址；不可据此宣称老师视景已接通。
