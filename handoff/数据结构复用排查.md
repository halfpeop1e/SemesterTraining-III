# 数据结构复用排查（防重复定义）

> 触发：用户指出"每个人负责的部分会有数据结构复用，可能出现重复定义；编写代码前先确认代码中是否已有可使用的数据结构"。
> 依据：本仓库内 [`数据结构依据摘录.md`](./数据结构依据摘录.md)（摘录自立项报告 §4.2/§4.3、小学期信号解析.md、704 司机台协议）+ `signal` 包现有代码。
> 原始文件（`.docx/...`）**不在 git 仓库内**（本地参考用，仓库内只读摘录）；下文表格已自含所需判定，clone 仓库即可查阅。
> 目的：写新类**之前**先查此表，已有的就复用/扩展，别另起炉灶。

## 1. 团队已约定的「统一数据对象」（立项报告 §4.3）

| 统一对象 | 内容 | 归属模块 | 代码现状 | 复用判定 |
|----------|------|----------|----------|----------|
| `line_profile` | 站点、距离、限速、坡度、停车点 | **轨道线路与信号控制（黄豪）** | ✅ 已有 `signal.model.LineProfile` + 17 个 model 类 + `LineProfileLoader` | **其他模块（动力学/停站/能源）一律复用 `LineProfile`，不得另建线路类**；按 `handoff/轨道线路与信号控制-交接文档.md` 调 `locateMileage/segStartM/segEndM` 查坐标 |

## 2. 三层数据结构对照（《小学期信号解析.md》）

### 第一层：静态线路数据（来自 Excel）—— ✅ 全部已有，归黄豪，他人复用
| 数据 | 现有类 | 备注 |
|------|--------|------|
| Seg 区段 | `TrackSegment` | 邻接链 + 长度 cm |
| 站台 | `Station` / `Platform` | |
| 限速 | `StaticSpeedRestriction` / `TemporarySpeedRestriction` | |
| 坡度 | `Gradient`（permille，真‰） | loader 已做 0.1‰→‰ |
| 应答器 | `Balise` | 静态位置 |
| 信号机 | `Signal`（含 `aspect`） | TASK-10 加 aspect |
| 道岔 | `Switch` + `SwitchState` | |
| 进路 | `Route` + `OverlapSection` | |
| 计轴区段 | `AxleCounterSection` | 含 `occupied` |
| 折返 | `Turnback` / `TurnbackType` | |

> **结论**：静态层无重复风险；任何模块要线路数据，**直接用 `LineProfile`**，不要重建 Seg/信号机/道岔等类。

### 第二层：车辆参数数据（`train_model`）—— 非黄豪，但与 `MaConfig` 有重叠风险
教学示例字段：`max_traction_acc / service_brake_acc / emergency_brake_acc / brake_response_time / max_speed_mps / train_length_m / jerk_limit`。

| 现有类 | 与 train_model 关系 | 复用判定 |
|--------|---------------------|----------|
| `signal.service.MaConfig` | **不是** train_model；是 MA 侧派生配置（`aBrakeMps2` 制动减速度、`tReactionS` 反应时间、`signalStopMarginM`、`maValiditySec` 等） | ⚠️ **风险点**：别把 `MaConfig` 当 train_model 复制。车辆模块应**自建** `train_model`（自己的包），MA 通过 `MaConfig` 消费其制动参数（由车辆/总控把 `service_brake_acc`、`brake_response_time` 填进 `MaConfig`）。两份对象、不同关注点，**不算重复**。 |
| `TrainState.lengthM` | 与 train_model 的 `train_length_m` 重叠 | 列车长度运行时取自 `TrainState.lengthM`（每车可不同）；train_model 是型号级默认。**运行时以 `TrainState.lengthM` 为准**，别在 MA 里再读 train_model 车长。 |

### 第三层：动态运行数据（列车实时状态）—— ⚠️ 重复定义高风险
教学示例：`train_id / current_seg_id / offset_m / speed_mps / acceleration_mps2 / traction_level / brake_level / mode / target_platform_id`。

| 现有类 | 字段 | 复用判定 |
|--------|------|----------|
| `signal.domain.TrainState` | `trainId / positionM(绝对里程m) / speedKmh / accelerationMps2 / lengthM / direction / timestamp` | ✅ **已解决（TASK-11）**：MA 的 `TrainState` 是"列车运行时状态"权威对象。教学示例用 `current_seg_id+offset_m`(段相对)+`speed_mps`，是另一种表示。**规则：不要新建第二个 TrainState**；其他模块用 `signal.service.TrainStateAdapter.fromSegmentRuntime(line, trainId, segId, offsetM, speedMps, dir, lengthM, ts)` 统一转换（段相对+m/s→绝对里程 m+km/h，走 `locateMileage`+`Units`）。非法 segId→`positionM=NaN`→`computeOne` 本车状态校验 fail-safe→`DEGRADED`。`traction_level/brake_level/mode/target_platform_id` 等车辆内部控制量是车辆模块私有，MA 不读，别塞进 `TrainState`。 |

## 3. 运行时事件结构对照（重复定义风险点）

| 教学示例结构 | 现有类 | 复用判定 |
|---------------|--------|----------|
| 区段占用 `{section_id, occupied, occupied_by}` | `AxleCounterSection`（有 `id/segIds/occupied`，**缺 `occupied_by`**） | ⚠️ 别另建"区段占用"类。需要"哪辆车占用"时，**扩展 `AxleCounterSection` 加 `occupiedBy` 字段**（非破坏性）。归黄豪，改前看交接文档红线。 |
| 信号机状态 `{signal_id, aspect, allow_pass, reason}` | `Signal`（含 `aspect:SignalAspect`）+ `SignalAspect.isProceed()`=allow_pass | ⚠️ 别用字符串 `"STOP"`。`aspect` 用 `SignalAspect` 枚举（协议 3.3.2 编码，`decodeSignalSystem(int)` 解码）；`allow_pass = aspect.isProceed()`。`reason` 若需要，**扩展 `Signal` 加 `reason` 字段**，别新建信号状态类。 |
| 进路状态 `{route_id, route_name, status, locked_by_train}` | `Route`（静态：`id/name/startSignalId/endSignalId/overlapIds`，**缺 status/locked_by_train**） | ⚠️ 别另建"进路状态"类。运行时可用性（AVAILABLE/LOCKED）需要时，**扩展 `Route` 加 `status/lockedByTrain`**，或总控维护独立的 `Map<routeId, RouteStatus>`（推荐后者，避免改静态类）。归周锦泽定。 |
| 应答器经过事件 `{train_id, balise_id, event, corrected_seg_id, corrected_offset_m}` | `Balise`（静态位置，非事件） | 不重叠。事件是车辆/引擎运行时产物，归车辆/引擎模块；`Balise` 静态类归黄豪，别混。 |
| 停站结果 `{train_id, platform_id, target/actual_position, stop_error, ...}` | `Platform`/`Station`（静态） | 不重叠。停站结果是评估/车载产物，归评估或车载模块。 |

## 4. 协议（704）消息字段 ↔ 现有类

| 协议字段（704） | 现有类 | 复用判定 |
|------------------|--------|----------|
| 列车 ID / 速度 / 位置 / 方向 / 长度 | `TrainState` | 复用；单位换算在边界（cm/s→km/h、cm→m、0x55/0xaa→Direction） |
| 列车载重 / 故障限速 / 定位丢失 / 完整性 / 可用制动数量 | `TrainState` **暂无** | ⚠️ 需要时**扩展 `TrainState` 加字段**（按交接文档"允许改动耦合点"），别新建列车类 |
| 信号机显示编码（3 套） | `SignalAspect`（取 3.3.2 为权威） | 复用 `decodeSignalSystem`；另两套由总控/视景各自解码 |
| 道岔状态 0x01/0x02/0x04 | `SwitchState`（NORMAL/REVERSE，**无四开**） | 扩 `FAIL` 枚举即可，别新建 |
| 方向 0x55/0xaa/0xff、0/1、+1/-1、float | `Direction` + `fromDriverConsole` | 复用；MA 为方向权威 |

## 5. 复用规则（写代码前必读）

1. **新建任何"列车/线路/信号/道岔/进路/计轴"类之前**，先查 `signal.domain` + `signal.model` 包，已有则复用或扩展，**不要平行新建**。
2. **`LineProfile` 及全部静态 model 归黄豪**，其他模块只读消费，不改结构（要加字段走交接文档"允许改动耦合点"）。
3. **`TrainState` 是运行时列车状态权威对象**（绝对里程 m + km/h）；段相对/mps 表示是模块内部私有，**进 MA 接口前必须用 `TrainStateAdapter.fromSegmentRuntime(...)` 转换**（已提供，别各写一份）。
4. **`train_model`/`power_profile` 不归黄豪**，由车辆/能源模块各自定义；`MaConfig` 是 MA 派生配置，不是 train_model，不重复。
5. **枚举一律用现有**（`Direction`/`SignalAspect`/`SwitchState`/`AuthorityBasis`/`SignalEvent`），别用字符串或魔法数；协议字节用对应 `decode/fromXxx` 转。
6. **单位换算只走 `util.Units`**，别在别处写 cm↔m、km/h↔m/s。
7. 扩展现有类（加字段）属"允许改动"，但改前看 `handoff/轨道线路与信号控制-交接文档.md` 红线，改后跑 verification。

## 6. 本模块已提供、可供复用的数据结构清单

- **领域对象**：`TrainState`、`MovingAuthority`、`Direction`、`SignalAspect`、`AuthorityBasis`、`SignalEvent`
- **线路 model（17 个）**：`LineProfile`、`TrackSegment`、`Switch`、`SwitchState`、`Signal`、`Turnback`、`TurnbackType`、`AxleCounterSection`、`OverlapSection`、`Route`、`Gradient`、`StaticSpeedRestriction`、`TemporarySpeedRestriction`、`Station`、`Platform`、`PhysicalSection`、`LogicalSection`、`Balise`
- **工具**：`util.Units`（单位换算）、`LineProfileLoader`（真实数据加载）
- **入口**：`MovingAuthorityService.compute(...)`、`Direction.fromDriverConsole(int)`、`SignalAspect.decodeSystem(int)` / `toDriverConsoleScreen()`

> 各模块若发现自己要建的类与上表重名/同义，**先停**，到 `handoff/` 看对应交接文档或联系模块负责人，确认是复用/扩展而非新建。
