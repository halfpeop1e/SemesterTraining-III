---

## 1. 模块范围

负责**线路/轨道静态建模**与**移动授权（MA）计算**：

- 线路建模：站点/区间/坡度/限速/道岔/折返/临时限速/计轴区段/进路/保护区段/信号机，支持真实 CBTC 数据加载。
- MA 计算：纯函数 `compute(...)` → 每列车 `{EoA 终点里程, 最大速度, basis, event}`；六维约束（前车占用/折返终点/道岔/信号机/计轴/进路+保护区段），UP/DOWN 双向镜像，坡度制动修正，fail-safe 默认收紧。
- 与司机台协议对齐：方向编码（`Direction.fromDriverConsole`）、信号机显示（`SignalAspect` + 到信号屏映射）。

**边界**：只算 MA + 建模。交付给谁、怎么传、`TrainState` 谁填、列车长度来源——不归本模块，由总控(周锦泽)/车载(郭逸晨)定。本模块只暴露 `MovingAuthorityService.compute(...)`。

## 3. 对外提供的接口

### 3.1 MA 计算（核心入口）
```java
package com.bjtu.railtransit.signal.service;
public class MovingAuthorityService {
    Map<String, MovingAuthority> compute(LineProfile line, List<TrainState> trains,
                                         Map<String, Route> routes, double nowSec);
    Map<String, MovingAuthority> compute(LineProfile line, List<TrainState> trains, Map<String, Route> routes);
    Map<String, MovingAuthority> compute(LineProfile line, List<TrainState> trains);
}
```
- **纯函数**，无副作用，不持有可变状态；同输入同输出；每 tick 调用一次。
- 返回 `Map<trainId, MovingAuthority>`；`MovingAuthority` = `{endOfAuthorityM(m), maxSpeedKmh(km/h), basis, event, capSignalId, routeId, timestamp}`。
- `event == DEGRADED || MA_EXPIRED` → 立即停车（`maxSpeedKmh` 此时 = `degradedSpeedKmh`，默认 0）。
- 获取实例：Spring 注入 `@Autowired`；非 Spring 手动 `new MovingAuthorityService(new MaConfig())`。

### 3.2 REST（松耦合/跨进程备选）
`POST /api/signal/ma`，入参 `{lineProfile, trains, routes?, nowSec?}`，返回 `ApiResponse{success,message,data}`，`data` = `Map<trainId, MovingAuthority>` 的 JSON。同进程优先直接调 `compute()`（更低延迟）。

### 3.3 给司机台/其他模块的工具方法
- `Direction.fromDriverConsole(int)`：司机台方向 `0=上行/1=下行/其余=INVALID` → `Direction`。
- `SignalAspect.decodeSignalSystem(int)`：协议 3.3.2 信号机编码字节 → `SignalAspect`（未知→null）。
- `SignalAspect.toDriverConsoleScreen()`：→ 司机台信号屏 红(0x01)/绿(0x02)/白(0x03)。
- `LineProfile.locateMileage(segId, offsetCm)` / `segStartM` / `segEndM`：坐标查询（调试/显示用）。

### 3.4 构造 TrainState 的权威入口（`TrainStateAdapter`，TASK-11 新增）
- `TrainStateAdapter.fromSegmentRuntime(line, trainId, segId, offsetM, speedMps, dir, lengthM, ts)`：**段相对 + m/s → 绝对里程 m + km/h**。其他模块喂 MA 前统一调它，不要各写转换、不要新建第二个 TrainState。
- `TrainStateAdapter.fromAbsolute(trainId, positionM, speedKmh, dir, lengthM, ts)`：已知绝对里程时的便捷入口。
- 非法 segId → `positionM=NaN` → `computeOne` 本车状态校验 fail-safe → `DEGRADED`。

## 4. 不可修改的核心代码（红线）

> 这些动了会破坏 fail-safe 语义或方向正确性。**联调时若需要改，必须先找黄豪确认**；其他模块不得擅自改。

1. `MovingAuthorityService.computeOne` 的 fail-safe 兜底顺序与六维候选收集逻辑：
   - `direction==null||INVALID` → degraded；`timestamp` 过期 → `MA_EXPIRED`；他车/前车状态未知 → `DEGRADED`。
   - 取「沿行车方向最近的前方边界」的 `min`(UP) / `max`(DOWN) 选取逻辑。
2. `TrackConstraintService` 六维 `eoaFromXxx` 的**方向感知实现**：
   - UP 取 `min` 且 `> 当前位`；DOWN 取 `max` 且 `< 当前位`；无约束返回 `+INF`（勿退回 `-INF`）。
   - 信号机停车余量：UP `m - signalStopMarginM`；DOWN `m + signalStopMarginM`。
3. `MovingAuthorityService.computeOne` 的 fail-safe 兜底顺序（方向→本车状态→过期→邻车未知→六维候选→min/max 选取）；**本车状态校验 `validState(self)` 不得删除**（含 adapter 非法 segId→NaN 场景）。
3. `TrackConstraintService.eoaFromSignals` 的**放行/截断规则**：`aspect != null && isProceed()`(绿灯) 不截断；`null`/停车/灭/断 截断（fail-safe）。
4. `TrackConstraintService.requiredGap` 坡度制动修正公式：`aEff = aBrakeMps2 + G*(permille/1000)`，`aEff = max(aEff, aFloorMps2)`。
5. 枚举值与映射：
   - `Direction` 的 `UP/DOWN/INVALID` 与 `fromDriverConsole` 映射。
   - `SignalAspect` 的 13 个枚举值、`decodeSignalSystem`、`isProceed`(仅 GREEN)、`toDriverConsoleScreen`(GREEN→绿/WHITE→白/其余→红)。
   - `AuthorityBasis` / `SignalEvent` 全部常量。
6. `util/Units` 所有换算函数（cm↔m、km/h↔m/s、0.1‰→‰）：任何数据换算**必须走 Units**，不得在别处另写魔法数。
7. `LineProfileLoader.enrich()` 的换算步骤（含坡度 0.1‰→‰ 第 7 步）：改了会让 MA 制动修正失真（已踩坑，见 changelog-9）。

## 5. 允许联调改动的耦合点

> 为对接其他模块**可以改**，但须遵守下列约束；改动后跑本模块自检确保不回归。

| 耦合点 | 可怎么改 | 约束 |
|--------|----------|------|
| `MaConfig` 参数值 | 调阈值（如 `maValiditySec`、`aBrakeMps2`、`signalStopMarginM`） | 不改字段语义；改后跑 verification 复验数值 |
| `Signal.aspect` 赋值 | 联锁侧把 `SignalAspect` 灌进 `Signal.setAspect(...)` | 用 `decodeSignalSystem` 解码，勿自造编码 |
| `Switch.state` 赋值 | 联锁侧灌 `SwitchState.NORMAL/REVERSE` | null/非 NORMAL 一律 fail-safe 截断，勿改此语义 |
| `TrainState` 新增字段 | 后续消费故障限速/定位丢失/完整性等 | 加字段+getter/setter；在 compute 消费时遵守 fail-safe（异常→收紧）；勿动既有字段语义 |
| `SwitchState` 扩枚举 | 补 `FAIL`(四开)/`UNKNOWN` | `NORMAL`/`REVERSE` 不变；新增值按"非 NORMAL → 截断"处理 |
| `LineProfileLoader` 新增数据源/换算步骤 | 适配新字段 | 换算**必须走 `Units`**；新步骤加到 `enrich()` 末尾 |
| `SignalController` 加端点 | 暴露查询/调试接口 | 不改 `compute(...)` 调用语义 |
| 新增 `model` 字段 | 非破坏性加字段 | 不改既有字段类型/语义 |
| 新增 verification/test | 加自检 | —— |

## 6. 依赖其他模块的输入（本模块需要别人喂什么）

| 输入 | 提供方 | 形式 | 备注 |
|------|--------|------|------|
| `TrainState`（每 tick） | 仿真引擎/总控 | `List<TrainState>` | `positionM`(m) / `speedKmh`(km/h) / `lengthM` / `direction` / `timestamp`(s) 必填且非 NaN；`lengthM>0` |
| `Route` Map | 总控(周锦泽) | `Map<trainId, Route>` | key 须与 `trainId` 一致；可空 Map |
| `Signal.aspect` | 联锁(周) | 灌入 `SignalAspect` | 未灌入(null)→fail-safe 硬截断 |
| `Switch.state` | 联锁(周) | `SwitchState` | null→fail-safe 截断 |
| `nowSec` | 引擎时钟 | `double`(s) | 须与 `TrainState.timestamp` 同基准 |
| `LineProfile` | 本模块加载/或总控 | `LineProfileLoader.loadFromJsonFile` | 须 `buildMileageIndex()` |

**坐标基准**：里程为绝对里程 m，原点=起点道岔(0)；`LineProfile` 须先 `buildMileageIndex()` 把 `(segId, offsetCm)` 链成绝对里程。

## 7. 单位约定（务必遵守）

| 量 | 单位 | 备注 |
|----|------|------|
| 速度 | km/h | 接口层；内部物理换算 m/s |
| 距离/位置/里程 | m | 真实 CBTC 数据常 cm，`LineProfileLoader` 自动换算 |
| 坡度 | ‰ | 负=下坡；真实数据以 0.1‰ 整数存，loader 已 ÷10 |
| 时间 | s | 仿真时刻、反应/制动时间 |
| 加速度 | m/s² | |

> 真实数据 cm/cm/s ↔ m/km/h 的换算**只在 `Units` + `LineProfileLoader` 边界做一次**，MA 内部统一 m/km/h。其他模块不得在 MA 内部塞 cm。

## 9. 联调对接步骤（给引擎/总控/车载）

1. 引擎 tick 循环里，在「总控更新 Route/信号/道岔**之后**、ATO**之前**」调 `compute(line, trains, routes, nowSec)`。
2. 总控把联锁状态灌进 `Signal.setAspect(...)` / `Switch.setState(...)`（用 `SignalAspect.decodeSignalSystem` 解码字节）。
3. 司机台方向字段经 `Direction.fromDriverConsole(...)` 解码后填 `TrainState.direction`。
4. ATO/司机台消费 `MovingAuthority`：`endOfAuthorityM` 作停车点、`maxSpeedKmh` 作限速；`event=DEGRADED/MA_EXPIRED` 立即制动。
5. 司机台信号屏显示用 `SignalAspect.toDriverConsoleScreen()` 取 红/绿/白 字节。

> 接入后若 `switch.state`/`signal.aspect` 不再为 null，MA 自动细化，**无需改本模块代码**。
