# handoff/ —— 模块交接文档

本文件夹用于**各模块之间的交接**，帮助负责其他模块的同学/agent 平滑接入、减少冲突。

## 约定

1. 每个模块开发完成后，由该模块负责人在本文件夹下放一份交接文档，命名：`<模块名>-交接文档.md`。
2. 交接文档至少包含：
   - **模块范围**与负责人
   - **覆盖的文件清单**（后端包/前端页面、关键源码与测试文件）
   - **对外提供的接口**（方法签名、入参/出参、单位、调用时机）
   - **不可修改的核心代码（红线）**：动了会破坏 fail-safe / 语义正确性的部分
   - **允许联调改动的耦合点**：为对接其他模块可以改、但须遵守约束的部分
   - **依赖其他模块的输入**：本模块需要别人喂什么
   - **验证方式**：怎么自测、沙箱编译命令
3. 跨模块改动前，先看对应模块的交接文档，**优先走"允许改动"的耦合点**，红线一律不动；改动后跑双方的自检。
